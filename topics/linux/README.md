### Что такое Linux

[▶️ Произношение — "Linux"](/data/mp3/linux.mp3)

[Wiki](https://ru.wikipedia.org/wiki/Linux): Linux (/ˈlɪnəks/ [ˈlɪnəks] или [ˈlɪnʊks], Ли́нукс; в части случаев GNU/Linux) — семейство Unix-подобных операционных систем на базе ядра Linux, включающих тот или иной набор утилит и программ проекта GNU, и, возможно, другие компоненты.
Linux-системы распространяются в основном бесплатно в виде различных дистрибутивов.

Есть минималистичные дистрибутивы на BusyBox, Alpine, Android, которые не включают GNU-утилиты, но в классическом десктопном/серверном понимании — почти всегда включают.

[RedHat](https://www.redhat.com/en/topics/linux/what-is-linux): Linux® - это операционная система с открытым исходным кодом (ОС), созданная Линусом Торвальдсом в 1991 году. Сегодня у неё огромная база пользователей, и она используется в 500 самых мощных суперкомпьютерах. Пользователи выбирают её по многим причинам, включая гибкость и высокий уровень безопасности.
Ядро Linux поддерживается международным сообществом разработчиков open-source, и существует сотни различных дистрибутивов Linux.

### sudo

#### что такое `sudo`

Большинство дистрибутивов задействуют пакет под названием `sudo`, чтобы позволить администраторам запускать команды от имени суперпользователя, если они вошли в систему от своего имени.

При выполнении данной команды sudo регистрирует это действие с помощью службы `syslog` в системном журнале для устройства `local2`.

### Файл `/etc/sudoers`

Конечно, система не позволяет любому пользователю выполнять команды от имени суперпользователя — вы должны настроить привилегии пользователей в  файле `/etc/sudoers`. Пакет sudo имеет множество параметров

> [!IMPORTANT]
> Редактируем только с помощью `visudo` проверяющей синтаксис перед сохранением, а не с помощью `vim`, `nano`, где ошибка может полностью заблокировать доступ к `sudo`.

Современный стандарт — не трогать основной файл, а создавать отдельные конфигурации в директории `/etc/sudoers.d/`

Типовые параметры:
- NOPASSWD: Настройка, позволяющая выполнять команды без ввода пароля. Это необходимо для скриптов автоматизации и CI/CD агентов (например, Jenkins или GitLab Runner)

Пример синтаксиса:
```bash
# Пользователь devops может перезапускать любые сервисы без пароля
devops ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart *

# Только конкретная команда
devops ALL=(ALL) NOPASSWD: /usr/bin/docker system prune -f
```
